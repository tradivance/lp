# MCP設定の留意事項

## Notion MCP Server

環境変数名に注意：
- ✅ 正しい: `NOTION_TOKEN`
- ❌ 間違い: `NOTION_API_KEY`

MCPサーバーのソースコードが `NOTION_TOKEN` を参照しているため、`NOTION_API_KEY` では認証エラー（401）になる。

設定例（.mcp.json）:
```json
{
  "mcpServers": {
    "notion-example": {
      "command": "npx",
      "args": ["-y", "@notionhq/notion-mcp-server@1.9.1"],
      "env": {
        "NOTION_TOKEN": "ntn_xxxxx"
      }
    }
  }
}
```

## トラブルシューティング

401エラーが出た場合のチェックリスト（確率順）：
1. 環境変数名が `NOTION_TOKEN` になっているか
2. APIキーが正しいか（curlで直接テスト）
3. Notionインテグレーションがページに接続されているか
4. 複数の設定場所（.mcp.json と ~/.claude.json）で重複していないか

## ライティングルール

### 絶対NG事項
- 構成は変えない
- 新しいHタグ（H1、H2、H3等の見出し）を追加しない
- 既存のHTML構造・ブロック順序を変更しない
- 絵文字は使わない
- 文字数を増やさない

### 段落ルール
- 1文ごとに個別のparagraphブロックで囲む
- 1文は80〜100文字程度

### 品質基準
- 読者が納得感と信頼感・親しみを感じられる内容にする
- 調査は直近6ヶ月〜1年以内の情報を使用
- 調査結果はファクトチェックする
- 出典がある場合は明記する

### トーン（重要）
- **ナレッジベース（石田社長の過去記事・ホワイトペーパー）の口調・表現・論理展開を参考にする**
- 石田社長と同じ語彙・言い回しを使う
- 専門用語は避けるか、必ず補足する

### ナレッジベース参照
- 記事執筆時はNotionのknowledgebase、または `knowledge/tradivance/` を参照
- 音声書き起こしは `knowledge/tradivance/transcripts/` に蓄積

### 音声書き起こしプロセス
1. 音声ファイルを準備（mp3, m4a, wav, mp4）
2. Whisper等で書き起こし → `transcripts/YYYY-MM-DD_タイトル_raw.md`
3. 整形・編集 → `transcripts/YYYY-MM-DD_タイトル.md`
4. 重要な知見は `knowledge/tradivance/` 直下にテーマ別ファイルとして抽出

## 記事タイプ

記事タイプは実行時にユーザーと対話で決定する。以下は調査の切り口と実行内容のテンプレート。

### pricing（料金・プラン比較）
- 値段、無料・有料の違い、具体的な料金プランと課金金額の目安を調査
- 調査期間: 直近1年以内

### howto-dev（開発方法・事例）
- 開発のやり方・始め方、開発事例、注意点、技術的リスクを調査
- 調査期間: 直近6ヶ月以内

### prompt（プロンプトの書き方）
- プロンプトの書き方、事例、出力コントロールの注意点・リスクを調査
- 調査期間: 直近6ヶ月以内

### compare（ツール比較）
- 複数ツールの機能・特徴・料金の違い、注意点・技術的リスクを調査
- 調査期間: 直近6ヶ月以内
- 変数: keyword1, keyword2, keyword3

### onboarding（導入手順・アカウント管理）
- 導入ステップ、アカウント管理、導入時の注意点・事例を調査
- 調査期間: 直近6ヶ月以内

### service（サービス定義・特徴）
- サービスの定義・特徴、導入事例、注意点、導入ステップを調査
- 調査期間: 直近6ヶ月以内

### generic（汎用）
- keyword × objectの組み合わせで調査
- 調査期間: 直近6ヶ月以内
- 変数: keyword1, object1

## 文体とトーン

### 基本的な文体

#### 親しみやすく実務的
- 堅苦しくない語調を採用
- 専門的でありながら人間的
- 単なるビジネスレポートではなく、関係者の想いを共有する語り口

#### 会話的・親密性重視
- 季節感に触れながら導入
- 「テナワンの小林です」という一人称で親近感を構築

### よく使われる表現・言い回し

#### 1. 関西弁的・方言的表現
- **「築古だからいいんやって」**（福井県方言に由来）
- 「やんちゃ坊主」という意味の「テナワン」

#### 2. 二項対立の問い
- 「築古ビル」vs「新築ビル」
- 「建て替えかリノベか」
- Before/Afterの対比

#### 3. 前向きな表現
- 「未来を考える」
- 「活路」
- 「生まれ変わる」

#### 4. 疑問形の使用
- タイトルに疑問形を用いる構成
- 読者の疑問に寄り添う形で展開

#### 5. 感情的な表現
- 「思わず涙が出そうに」
- 「心の奥が熱くなりました」
- 「至福のひととき」

#### 6. 視覚的・感覚的な表現
- 「絶妙なからし色」
- 「まるで別物になりました」
- Before/Afterの写真で変化を明��

#### 7. 地道さへの賞賛
- 「いくつもの小さな工夫を積み重ねて、その地道さが、仕上がりの美しさを決めます」
- 細部へのこだわりを価値づけ

### トーンの特徴

#### 1. ビルオーナー・経営者への寄り添い
- 「ビルオーナーに知ってほしい」
- 実務的課題を抱える経営層向けの語りかけ

#### 2. 実体験ベース
- 実際に訪問した経験を共有
- 具体的な描写で読者への親近感を構築

#### 3. 建築・不動産の専門性と親しみやすさの両立
- 専門用語は使うが、必ず補足する
- 複雑な不動産テーマを分かりやすく伝える

#### 4. 人間味のある表現
- テナントの想いや背景を丁寧に紹介
- 引用符で当事者の言葉を大切にする
